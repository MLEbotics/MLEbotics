3. Voice + Alerting (Key Investor Differentiator)

Voice Commands Supported

Stop, speed up, slow down, pause, resume, distance, pace, status, open box, close box, play music, alert ahead, clear left, clear right, return to start.

People-Ahead Alert Flow

  1. Runner taps alert button or says a trigger phrase.
  2. Robot speaker announces runner is approaching.
  3. App opens SMS to each saved contact with distance/ETA/message.

Directional Messaging

  • Clear Left: asks people to move right
  • Clear Right: asks people to move left
  • Default: generic polite clear-route request

4. Robot Behavior + Speed Logic

Condition Robot Response
Runner 0–5 m behind Maintain full target pace
Runner 5–8 m behind Reduce to 60% of target pace
Runner >8 m behind for 3+ seconds Stop; show runner-lost status in app
Obstacle <50 cm ahead Stop, servo sweep, choose clearer side
Both sides blocked Reverse slightly and rescan
Waypoint complete Stop and mark workout complete
Speed framework: target pace is runner-selected; firmware and app support up to 30 km/h; speed can be adjusted by voice and app controls during a run.

6. App Architecture

Approach Pros Cons Fit
Fully Native Flutter Best device integration, low-latency control Store release required for major updates Excellent for robot control, BLE, safety loops
WebView-heavy Fast cloud updates Weaker hardware integration and performance Not ideal for real-time control
Hybrid cloud modules (current approach) Better performance; cloud-driven UI updates; store-friendly Higher architecture complexity Best middle-ground at scale

Recommended Plan

Keep robot control, BLE, safety logic, and voice pipeline in stable app binaries. Move non-critical UI modules and content surfaces to cloud-loaded modules over time.

Section 3 of 3 — Technical Project Plan

1. Project Overview

Running Companion is a mobile + hardware system that lets a runner set a target pace, then have a small 4-wheeled robot physically lead them on an outdoor GPS route — slowing when the runner falls behind, navigating around obstacles, and stopping if the runner gets too far away. Run data syncs to Firebase so history is available across all devices.

Mobile App

  • Flutter (Android / iOS / Web / Windows)
  • Firebase email/password + Google login
  • Pace Workout screen — live stats + waypoints
  • Run history saved to Firestore per user
  • Connects to robot via WiFi Direct (HTTP)
  • Stripe subscription (Free + Premium tiers)

Robot

  • Freenove 4WD ESP32 car kit
  • NEO-6M GPS for navigation & speed
  • SG90 servo sweeps ultrasonic sensor
  • Avoids obstacles, stops if runner lost
  • Creates its own WiFi hotspot

4. Robot Behaviour Logic

Situation Robot Response
Runner 0–5 m behind Full target pace
Runner 5–8 m behind Slow to 60% of target pace
Runner >8 m behind for <3 s Countdown timer starts
Runner >8 m behind for ≥3 s Stops completely — "Runner Lost" in app
Runner comes back within 8 m Immediately resumes pacing
Obstacle <50 cm ahead Stops, sweeps servo left & right
One side clear (>80 cm) Turns and drives around obstacle
Both sides blocked Reverses slightly, re-scans
Final waypoint (within 3 m) Stops, marks workout complete

5. Battery Life

The Freenove kit uses 2× 18650 cells. With ESP32 WiFi hotspot, 4 motors at running pace, servo, and 2 ultrasonic sensors — draw is roughly 1.1–1.5 A.

Battery Capacity Runtime
Cheap no-name 18650 ~1500 mAh ~30–40 min
Samsung 25R / LG HG2 2500–3000 mAh 50–75 min

Speed is measured via GPS Doppler — no wheel encoders needed. Accurate to ±0.1 km/h at running speeds.

6. Firmware Flash Steps

  1. Install Arduino IDE 2 (arduino.cc)
  2. Add ESP32 board URL in File → Preferences:
    https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json
  3. Install libraries: TinyGPS++, ArduinoJson, ESP32Servo
  4. Open robot_firmware/runner_pacer/runner_pacer.ino
  5. Board: ESP32 Dev Module, correct COM port
  6. Click Upload ⬆
