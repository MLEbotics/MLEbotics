1 · Project Overview

Running Companion is a mobile + hardware system that lets a runner set a target pace, then have a small 4-wheeled robot physically lead them on an outdoor GPS route — slowing when the runner falls behind, navigating around obstacles, and stopping if the runner gets too far away. Run data is synced to Firebase so history is available across devices.

Mobile App

  • Flutter (Android / iOS / web)
  • Firebase email/password login
  • Pace Workout screen — ar stats
  • Run history saved to Firestore
  • Connects to robot via WiFi Direct (HTTP)

Robot

  • EMO 4WD Robot Chassis with speed encoders + ESP32 WROOM-32
  • NEO-6M GPS for navigation & speed
  • SG90 servo sweeps ultrasonic sensor
  • Avoids obstacles, stops if runner lost
  • Creates its own WiFi hotspot

3 · Immediate Next Steps

Step A — Configure Firebase (unblocks login + run history)

  1. Go to console.firebase.google.com → Create project → name it runner-companion
  2. Authentication → Sign-in method → enable Email/Password
  3. Firestore Database → Create database → start in test mode
  4. Add an Android app: package name com.example.runner_companion_app
  5. In a terminal inside the project folder:
dart pub global activate flutterfire_cli
flutterfire configure

This replaces lib/firebase_options.dart with real credentials. Then rebuild:

$env:PATH = "$env:JAVA_HOME\bin;$env:PATH"
flutter build apk --release

Step C — Flash the Robot

  1. Install Arduino IDE 2 (arduino.cc)
  2. Add ESP32 board: File → Preferences → add https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json
  3. Install libraries via Sketch → Include Library → Manage Libraries:
    • TinyGPS++ by Mikal Hart
    • ArduinoJson by Benoit Blanchon
    • ESP32Servo by Kevin Harrington
  4. Open robot_firmware/runner_pacer/runner_pacer.ino
  5. Select board: ESP32 Dev Module, correct COM port
  6. Upload ⬆

5 · Robot Behaviour Logic

Situation Robot Response
Runner 0–5 m behind Full target pace
Runner 5–8 m behind Slow to 60% of target pace
Runner >8 m behind for <3 s Countdown timer starts
Runner >8 m behind for ≥3 s Stops completely — "Runner Lost" shown in app
Runner comes back within 8 m Immediately resumes pacing
Obstacle <50 cm ahead Stops, sweeps servo left & right
One side clear (>80 cm) Turns and drives around obstacle
Both sides blocked Reverses slightly, re-scans
Cleared obstacle Returns to GPS navigation
Final waypoint reached (within 3 m) Stops, marks workout complete

6 · Battery Life

The robot runs on a 3S 11.1V LiPo battery (2200–3000mAh). With ESP32 WiFi hotspot, 4 motors at running pace, and 3 ultrasonic sensors the draw is roughly 1.1–1.5 A.

Battery Capacity Runtime
3S LiPo 2200mAh 2200 mAh ~45–55 min
3S LiPo 3000mAh 3000 mAh 65–80 min

Speed is measured via GPS Doppler — no wheel encoders needed. Accurate to ±0.1 km/h at running speeds.
