Running Companion

AI-powered pacing robot for safer, smarter outdoor training.

Investor Pitch • Short Version
Prepared February 27, 2026

1) Problem

Outdoor runners train alone without real-time pacing support, safe crowd signaling, or interactive coaching that adapts live to conditions.

Pain Points

  • Pace drift and inconsistent training quality
  • Unsafe path interactions in crowded routes
  • No hands-free way to control devices while running

Market Gap

  • Wearables track data but do not physically pace
  • Coaching apps are mostly passive and reactive
  • No integrated robot + app solution in this category
Slide 2

2) Solution

A small ESP32 robot physically leads the runner while a Flutter app provides pace control, voice interaction, and AI coaching.

Core Experience

  • Set target pace and route waypoints
  • Robot leads and adapts to runner distance
  • Obstacle detection and automatic safe behavior

Connected Features

  • Garmin BLE telemetry: HR, cadence, speed, distance
  • Voice commands: stop, speed, resume, return
  • AI coach with spoken feedback during runs

5) 🧩 Hybrid Frameworks Loading Cloud Modules

This is a middle ground: a native app that loads UI or logic from the cloud.

How it Works

  • The app is built with Flutter, React Native, or Ionic.
  • Most screens or logic are fetched from the cloud.
  • The app updates itself by pulling new modules from your server.

Pros

  • Better performance than WebView.
  • Still supports cloud-based updates.
  • App stores accept these more easily.

Cons

  • More complex architecture.
  • You must design a module-loading system.

Examples

  • Tesla app
  • Airbnb app (older versions)
  • Many enterprise apps
